1. Partly true, if we are talking about the frequent rewriting of the same key, the cache loses its meaning. But what if your workload involves writing many different keys? In this case, you should think about write optimization.

2. Of course, reading will be a little slower for such a cache. The repository has benchmarks for reading workfloads. And the partitioned map is 17% slower in this case. This is a tradeoff for 315% faster writing.
